Page Builder

AI 채팅으로 HTML 페이지를 생성하고, 저장·공유할 수 있는 웹 애플리케이션

목적

사용자가 원하는 UI 화면을 자연어로 설명하면, 로컬 AI 모델(Ollama)이 실시간으로 Bootstrap 기반 HTML 페이지를 생성합니다. 생성된 페이지는 고유 URL로 저장·공유됩니다.

무료 & 로컬

외부 유료 API 없이 Ollama를 로컬에서 실행하여 사용합니다. 생성된 모든 데이터는 내장 H2 데이터베이스에 저장됩니다.

URL 구조

URL설명
/login로그인 페이지
/register회원가입 페이지
/dashboard내 페이지 목록 + 업로드된 파일 관리
/editor새 페이지 생성 (AI 채팅)
/editor/{id}기존 페이지 수정

/docs이 문서 페이지
/h2-consoleH2 DB 콘솔 (개발용)

개발 진행 현황

1단계 — 기본 인프라 구축 완료
Spring Boot 2.7.18 프로젝트 생성, 포트 9998, H2 내장 DB + JPA, Spring Security 5 (BCrypt, 세션 기반 인증), 회원가입/로그인/로그아웃 구현.
해결 이슈: 순환 참조 (PasswordEncoderConfig 분리), 한글 쿠키 RequestRejectedException (StrictHttpFirewall 완화)
2단계 — 대시보드 & 페이지 관리 완료
내 페이지 목록(카드 + iframe 썸네일), 페이지 저장 시 UUID 공개 URL 발급, 페이지 삭제 및 수정 기능.
3단계 — AI HTML 생성 (Ollama 연동) 완료
Ollama REST API (codellama 모델), Pre-fill 기법으로 HTML 즉시 생성 강제, 분할 화면 에디터 (채팅 + iframe 미리보기), Bootstrap/FontAwesome 자동 포함 시스템 프롬프트.
해결 이슈: codellama가 HTML 대신 설명 텍스트 반환 → assistant pre-fill 메시지 추가로 해결
4단계 — 채팅 히스토리 & 품질 개선 완료
수정 모드 진입 시 이전 대화 내역 표시, 대화 컨텍스트 유지하며 추가 수정 가능, 시스템 프롬프트 강화 (디자인 기준, 컴포넌트 가이드, 페이지 타입별 템플릿).
5단계 — 파일 참고 기능 (PPT/Excel/이미지) 완료
Apache POI 5.2.5로 PPT(PPTX/PPT)·Excel(XLSX/XLS) 텍스트 추출, 이미지(PNG/JPG) base64 저장 후 llava 비전 모델 전달, 에디터 내 📎 첨부 버튼 + 이미지 썸네일 미리보기.
6단계 — SSE 스트리밍 & 진행 현황 표시 완료
HttpURLConnection 기반 NDJSON 스트리밍 → Spring SseEmitter로 클라이언트 전달, 실시간 경과 시간 + 토큰 카운터, 생성 완료 후 작업 요약 카드 (모델명·처리시간·토큰수·참고파일), 내 파일 재사용 드롭업.
해결 이슈: SseEmitter 180초 타임아웃 → 0(무제한)으로 변경, IllegalStateException 처리

시작하는 방법

1. 사전 준비

필수 조건
· Java 17 (Eclipse Adoptium JDK 17 권장)
· Apache Maven
· Ollama 설치 및 실행 중 (ollama serve)
· codellama 모델: ollama pull codellama

2. 서버 실행 (PowerShell)

cd e:\claude관련\claude_test_project\projects\page-builder
$env:JAVA_HOME = "C:\Program Files\Eclipse Adoptium\jdk-17.0.8.101-hotspot"
mvn spring-boot:run

서버가 시작되면 http://localhost:9998에서 접속합니다.

3. 첫 사용 플로우

  1. 회원가입 — /register에서 아이디·이메일·비밀번호 입력
  2. 로그인 — /login에서 로그인 후 대시보드로 이동
  3. 새 페이지 만들기 — 대시보드에서 "새 페이지 만들기" 클릭
  4. 채팅으로 생성 — 에디터에서 원하는 화면 설명 입력 후 Enter 또는 전송 버튼
  5. 저장 — 결과가 마음에 들면 "저장" 버튼 → 제목 입력 → 공개 URL 발급

에디터 사용 가이드

채팅 패널 (왼쪽)
  • 자연어로 원하는 화면 설명 입력
  • Enter = 전송, Shift+Enter = 줄바꿈
  • 📎 버튼: 새 파일 업로드
  • 📁 버튼: 이전에 업로드한 파일 재사용
  • 이전 대화 내역 표시 (수정 모드)
미리보기 패널 (오른쪽)
  • 생성된 HTML 실시간 렌더링
  • 로딩 오버레이: 경과 시간 + 토큰 수
  • 상단 입력창: 페이지 제목 수정
  • 저장 버튼으로 공개 URL 발급

효과적인 프롬프트 작성 팁

잘 되는 요청 예시
· "관리자 대시보드 만들어줘 — 상단에 통계 카드 4개, 하단에 최근 주문 테이블"
· "로그인 페이지 — 이메일, 비밀번호 입력, 소셜 로그인 버튼 포함"
· "사용자 목록 페이지 — 검색바, 필터, 페이지네이션 있는 테이블 형태"
주의
너무 짧은 요청(예: "페이지 만들어")은 모델이 최소한의 HTML만 생성할 수 있습니다. 구체적으로 구성 요소를 나열할수록 좋은 결과가 나옵니다.

수정 요청하기

페이지가 저장된 후 /editor/{id}로 돌아오면 이전 대화 내역이 표시됩니다. 이어서 채팅으로 수정 요청을 하면 이전 컨텍스트를 유지한 채 수정합니다.

예: "헤더 색상을 파란색으로 바꿔줘", "테이블에 검색 필터 추가해줘", "모바일 화면에서도 잘 보이도록 반응형으로 수정해줘"

llava 모델 필요

파일 사용 방법

  1. 에디터 입력창 하단의 📎 버튼으로 파일 업로드 (또는 📁 버튼으로 이전 파일 선택)
  2. 파일이 트레이에 칩(chip)으로 표시됨. 이미지는 썸네일로 미리보기 가능
  3. 메시지 입력 후 전송 — 파일과 메시지가 함께 AI에 전달됨
  4. 이미지 파일이 포함된 경우 자동으로 llava 비전 모델로 전환
PPT 시각적 재현 최선의 방법
PPT 파일을 직접 올리면 텍스트만 추출되어 레이아웃 재현이 불완전합니다.
PPT 슬라이드를 스크린샷(PNG/JPG)으로 저장 → 이미지로 업로드하면 llava 비전 모델이 실제 레이아웃, 색상, 구성을 그대로 분석하여 재현합니다.

내 파일 재사용 (📁 버튼)

한번 업로드한 파일은 다시 업로드하지 않아도 됩니다. 📁 버튼을 클릭하면 이전에 업로드한 파일 목록이 나타나며, 클릭하면 즉시 첨부됩니다. 대시보드에서도 업로드된 파일 목록을 확인할 수 있습니다.

스트리밍 진행 현황

HTML 생성 요청을 보내면 Ollama가 토큰을 생성할 때마다 SSE(Server-Sent Events)로 실시간 전달됩니다.

📡 스트리밍 중 표시
  • 로딩 오버레이 + 스피너
  • 사용 모델명 (status 이벤트)
  • ⏱ 경과 시간 (1초마다 갱신)
  • 📝 생성된 토큰 수 (실시간)
✅ 완료 후 표시 (작업 요약 카드)
  • 🤖 사용한 AI 모델명
  • ⏱ 총 처리 시간
  • 📝 총 생성 토큰 수
  • 📎 참고한 파일 목록

SSE 이벤트 흐름

POST /api/chat/stream
    ↓
event: status   → 모델명 등 상태 메시지
event: token    → HTML 조각 (반복, 수백~수천 회)
event: complete → { html, tokenCount, modelName, elapsedMs, sessionId }
    ↓
에러 시: event: error → 오류 메시지
생성 시간 안내
codellama는 CPU만 사용할 경우 HTML 한 페이지 생성에 3~10분이 걸릴 수 있습니다. 타임아웃이 제거되어 완료될 때까지 기다립니다. 타이머가 올라가고 있으면 정상 진행 중입니다.

AI 모델 안내

codellama 기본 모델
  • 텍스트 기반 HTML/코드 생성
  • 텍스트·PPT·Excel 파일 참고 시 사용
  • 설치: ollama pull codellama
HTML 생성 시 pre-fill 기법 사용: AI가 <!DOCTYPE html>부터 바로 이어 생성하도록 강제.
llava 비전 모델
  • 이미지 분석 + HTML 생성
  • 이미지 파일 첨부 시 자동 전환
  • 설치: ollama pull llava
미설치 시 이미지 첨부 후 오류 발생. llava 설치가 필요합니다.

모델 설정 변경 (application.yml)

ollama:
  base-url: http://localhost:11434
  model: codellama        # 텍스트 기반 HTML 생성 모델
  vision-model: llava     # 이미지 분석 모델
  timeout: 120000

알려진 이슈 & 팁

⬜ 화면이 흰색으로 나옴

codellama가 HTML 구조는 생성하지만 body 내용이 부족한 경우. 또는 생성 도중 오류 발생.

→ 채팅창에 오류 메시지를 확인하세요.
→ 더 구체적인 요청으로 다시 시도하세요.
→ 타이머가 올라가고 있으면 아직 생성 중입니다 (3~10분 소요 가능).
⏱ 생성이 너무 느림

codellama는 CPU 기반으로 느릴 수 있습니다.

→ GPU가 있는 경우 Ollama가 자동으로 GPU를 사용합니다.
→ application.yml에서 더 작은 모델로 교체 가능 (예: model: tinyllama).
🖼️ 이미지 분석이 안 됨

llava 모델이 설치되어 있지 않은 경우.

→ PowerShell에서: ollama pull llava
→ llava는 용량이 크므로 (4~7GB) 다운로드에 시간이 걸립니다.
🔧 서버 실행 시 Java 버전 오류
UnsupportedClassVersionError: class file version 61.0
→ PowerShell에서 JAVA_HOME을 Java 17로 설정 후 실행:
$env:JAVA_HOME = "C:\Program Files\Eclipse Adoptium\jdk-17.0.8.101-hotspot"
🏠 H2 DB 초기화

개발 중 DB를 초기화하고 싶은 경우.

→ /h2-console에서 직접 테이블 조회/삭제 가능
→ JDBC URL: jdbc:h2:file:./data/pagebuilder
